# documents/

## What is this?

このフォルダはプロジェクトのドキュメントを一元管理する場所です。
仕様書・設計書の格納先ルールとフォルダ構成を定義します。

仕様書・設計書を管理するフォルダです。

## フォルダ構成

```
documents/
├── README.md       # このファイル（フォルダの使い方）
├── spec/           # 仕様書
└── design/         # 設計書
```

## ルール

- 仕様書は `spec/` に保存する
- 設計書は `design/` に保存する
- ファイル名は内容が分かる名前にする（例: `api-spec.md`, `db-design.md`）
- 各ドキュメントの冒頭には `## What is this?` セクションを設け、本ドキュメントの目的と記載対象のスコープを簡潔にまとめること
